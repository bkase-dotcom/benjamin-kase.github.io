ML5.js Inventory System

A three-layer computer vision pipeline that combines COCO-SSD object detection, Teachable Machine classification, and centroid tracking to maintain persistent object identity across video frames — all running in the browser.

The Problem

Off-the-shelf object detection (COCO-SSD) identifies what objects are in a frame, but treats each frame independently. It can't distinguish between two instances of the same object class, and it has no memory across frames. For an inventory system, you need to know not just "there's a bottle" but "this is the same bottle that was here 30 frames ago, and it's a specific type of bottle."

Three-Layer Architecture

Each layer solves a specific limitation of the layer below it, building from generic detection to persistent, classified identity.

Layer 1: COCO-SSD Detection

Pre-trained object detection provides bounding boxes and generic class labels (e.g., "bottle", "cell phone"). Runs per-frame with no cross-frame memory. This is the foundation — fast, reliable, but generic.

Layer 2: Teachable Machine Classification

A custom-trained classifier runs on cropped bounding box regions to provide specific labels (e.g., "Nalgene water bottle" vs. "glass jar"). Teachable Machine enables rapid retraining for new inventory items without code changes.

Layer 3: Centroid Tracking

Centroid-based tracking assigns persistent IDs by matching detection centroids across frames using distance thresholds. This solves the identity problem — even when detection flickers, the tracker maintains continuity.

Pipeline Flow

Each video frame flows through all three layers sequentially: COCO-SSD produces bounding boxes → Teachable Machine classifies each box → centroid tracker matches detections to existing tracked objects or creates new IDs. The output is a persistent, classified inventory of all visible objects, updated in real time.

What It Means

This project demonstrates that meaningful computer vision applications don't require a single monolithic model. By composing three lightweight, specialized layers — each solving one specific problem — you get a system that's more flexible, more debuggable, and more adaptable than an end-to-end approach. The architecture is the insight: detection, classification, and tracking are separable concerns.
